# Contributing

## Workflow
- Create a new branch from `develop`
- Use feature branches (feature/...) for new changes
- Keep changes focused on a single task

## Pull Requests
- Open a pull request to `develop`
- Provide a clear title and description
- Link the PR to an issue if possible
- Ensure the code runs and is understandable

## Review Process
- At least one team member reviews the PR before merging

## Note
From now on, all changes should follow the feature branch workflow and be merged via pull requests.
